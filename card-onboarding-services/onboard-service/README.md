# Onboard Service

The **Onboard Service** is the central orchestrator and state-machine of the Card Onboarding Platform.

## 🧭 Responsibilities
* Accepts single onboarding request payloads from `card-onboarding-worker`.
* Queries step execution states stored in `onboard-service-request-status`.
* Resumes transactions from where they failed.
* Requests customer registration on `customer-management-service`.
* Requests interest rates information on `account-management-service`.
* Writes completion records to `onboard-service-account-details`.

## 📁 Layout
* `swagger-internal.yaml`: OpenAPI contract.
* `pkg/onboard`: Autogenerated server interfaces and clients.
* `internal/orchestration`: Core state logic.
* `internal/store`: DynamoDB interface repository.
